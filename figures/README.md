# Presentation figures ‚Äî Gray-Scott temporal JEPA (The Well)

Two notebooks that produce talk-quality figures for the Gray-Scott / The Well
track. All images are written to `figures/out/` as both `.png` (dpi 200) and
`.svg`.

```
figures/
  plot_style.py              shared style + data access + analysis helpers
  01_dataset_overview.ipynb  figures of the original dataset
  02_results.ipynb           figures of training + rollout results
  out/                       generated images (PNG + SVG)
```

## 1. `01_dataset_overview.ipynb` ‚Äî the dataset

Field snapshots, temporal montage, a regime gallery, value distributions vs the
z-score stats, a radial power spectrum, the temporal change-rate that motivates
`time_stride`, and an optional animated GIF.

Data access is automatic, in this order (set `DATA_ROOT` / `SPLIT` in the config
cell):

1. **Local HDF5** ‚Äî if `DATA_ROOT/data/<split>/*.hdf5` exists (the layout of
   `eb_jepa/datasets/gray_scott/dataset.py`), read `t0_fields/{A,B}` directly.
2. **HuggingFace streaming** ‚Äî else stream a few trajectories of
   `polymathic-ai/gray_scott_reaction_diffusion` via `the_well.data.WellDataset`
   (no multi-TB download).
3. **Synthetic fallback** ‚Äî else run a tiny built-in Gray-Scott solver so the
   notebook still renders real reaction-diffusion patterns offline.

### Download real data by id (per-file)

Gray-Scott stores **one regime per HDF5 file** (6 files per split), and the files
are large (valid/test ~2.65 GB each, train ~21 GB each), so you pick files **by
id** rather than pulling a whole split. This uses `huggingface_hub` only
(torch-free); no `the_well` install needed. Files land at
`datasets/the_well/<dataset>/data/<split>/` (gitignored), which is a ready-to-use
`DATA_ROOT`.

In Notebook 1, set `DATASET` / `SPLIT` and `FILE_IDS` in the config cell; the
listing + download cells show the id table and fetch your selection. Or from a
shell:

```bash
# list files + ids (no download)
.venv/bin/python figures/plot_style.py --dataset gray_scott_reaction_diffusion --split valid --list
# download files 0 and 4 -> prints DATA_ROOT
.venv/bin/python figures/plot_style.py --dataset gray_scott_reaction_diffusion --split valid --download 0 4
```

Helpers: `plot_style.list_remote_files(dataset, split)` (returns
`{id, filename, regime, F, k, size_mb}`), `plot_style.format_remote_listing(...)`,
and `plot_style.download_well_files(dataset, split, ids, base_dir=...)` (returns
`DATA_ROOT`).

## 2. `02_results.ipynb` ‚Äî the results

Consumes artifacts produced by `eb_jepa/experiment_logger.py` and the baseline
evaluator. Set `RUN_DIR` (an `ExperimentLogger` run folder) and `BASELINE_DIR`
(a `examples/gray_scott/baselines` output folder) in the config cell. Every
section degrades gracefully if an artifact is missing.

Inputs it reads:

| Source | File | Figure |
|--------|------|--------|
| `ExperimentLogger` | `losses_step.csv` | training loss curves (total / vc / pred) |
| `ExperimentLogger` | `losses_epoch.csv` | train vs val loss |
| `ExperimentLogger` | `predictions/latent_metrics.json` | latent MSE vs horizon per epoch |
| `ExperimentLogger` | `predictions/field_metrics.json` | field MSE vs horizon (if present) |
| `ExperimentLogger` | `predictions/latent_epoch{E}.png` | embedded latent panel |
| baselines | `per_model_rollout_vrmse.csv` | **JEPA vs FNO/U-Net/persistence** VRMSE vs horizon |
| baselines | `metrics.csv` | window VRMSE (one-step / 6:12 / 13:30) bar chart |
| baselines | `rollout_comparison.png` | embedded rollout filmstrip |

A JEPA decoded-rollout VRMSE curve can be dropped into the headline figure as
soon as `examples/gray_scott/eval.py` (`build_decoder` / `vrmse_per_horizon`) is
implemented ‚Äî point `JEPA_ROLLOUT_CSV` at a `horizon,vrmse` CSV.

## Dependencies

`numpy`, `matplotlib`, `pandas`, `h5py`, `imageio` (for the GIF), and
`huggingface_hub` (for the id-based download). `the_well` is optional and only
used for HF streaming. Notebook 2 needs none of `torch` / `the_well`. The `out/`
folder is generated output and can be regenerated by re-running the notebooks.
